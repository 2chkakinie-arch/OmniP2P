OmniP2P とは

OmniP2P は、ブラウザだけで完結する P2P 通信・永続化ライブラリです。 Nostr リレーをシグナリング/フォールバック搬送に、WebRTC を主搬送に、 Blossom / NIP-96 / IPFS ゲートウェイをコンテンツアドレス永続化ストレージに使い、 さらに E2EE (X25519 + AES-GCM) と CRDT 共同編集 (MiniCRDT, Yjs 互換) までを 単一 JS ファイルに詰め込んでいます。

サーバー不要・ビルド不要・依存パッケージ不要。 <script src="v3.js"> を書くだけで、 チャット・音声/動画通話・画面共有・録画保存・共同編集・KV データベースを備えたアプリを作れます。

💡 コンセプト: 「Nostr の永続と可達性」+「WebRTC の低遅延」+「Blossom の分散ストレージ」を ひとつの API 面に統合し、開発者はプロトコルを意識せず普通のイベント駆動 API として使えます。

主要機能

🔌

Nostr → WebRTC 自動移行

決定的イニシエータ選出 + ICE restart で確実にネゴシエート。P2P 確立後は超低遅延。

📡

自動フォールバック

WebRTC が張れない相手には、Nostr ブロードキャストへ自動で切替え。必ずメッセージが届きます。

💾

Blossom / NIP-96 永続化

コンテンツアドレスな KV + Blob DB。書き込みは 1 本高速、裏で非同期に多重冗長化。

🎥

音声/動画通話 & 録画

startCall() / shareScreen()、MediaRecorder で暗号化保存 + Nostr 告知。

🔒

E2EE 標準搭載

X25519 ECDH → HKDF → AES-GCM 256 (AAD)。保存前にも暗号化してからアップロード可能。

🧬

CRDT 共同編集

MiniCRDT (Yjs 互換シェイプ)。WebRTC で即時同期、Nostr で永続化。

🧩

プラガブル拡張

Yjs / Automerge / Matrix / GunDB / OrbitDB / Iris / Waku を 必要な時だけ 遅延ロード。

⚡

ゼロ依存 / ゼロビルド

純 JS 実装の secp256k1 (BIP-340 schnorr) + X25519。<script> だけで動く。

🛡️

放置切断バグ対策

EOSE 応答実確認 heartbeat + 実イベント watchdog + スタガー再接続で瞬断ゼロ。

v3 での変更点 (WHAT'S NEW)

v3 ではコアを「軽量な通信・保存エンジン」に絞り、重い外部依存 (Gun / Matrix / Yjs / Waku 等) は 必要な時だけ CDN から遅延ロードする外部アダプタに分離しました。

🎯 レビュー全指摘 (#1〜#12) への対応

#修正内容
#1ハートビートは EOSE 応答を実確認してから「生存」とみなす (half-open 検知)
#2watchdog は アプリイベント受信のみでリセット (ハートビート応答では ×)
#3リレー再接続をスタガー + 「最低 1 本維持」フェイルセーフで瞬断ゼロ化
#4Room.send のフォールバック判定を pc.open 実数ベースに統一
#5SDP/ICE 等の重要イベントは minAck 複数 + ack 不足時の自動再送
#6保留 ICE 候補にリングバッファ上限 (重複配送の氾濫を抑制)
#7polite 側も restart タイムアウト後に re-offer 要求へフォールバック
#8RelayConnection と PeerConnection の再接続を状態機械で協調 (glare 低減)
#9DataChannel bufferedAmount バックプレッシャー流量制御
#10再接続時に since を「最後に受信した時刻」に更新し取りこぼし防止
#11RTC が connected なら presence timeout だけでは切断しない (RTC 優先)
#12ハートビート専用サブ ID を再接続毎に再生成

インストール

1. CDN から (推奨)

<script src="https://cdn.jsdelivr.net/gh/2chkakinie-arch/OmniP2P@main/v3.js"></script>

ブラウザでは window.OmniP2P が使えるようになります。

2. Node / CommonJS

const OmniP2P = require('./v3.js');

3. AMD

UMD ラッパーが同梱されているため AMD ローダーからも define(['OmniP2P'], fn) で使えます。

4. ネイティブ ESM

ファイル末尾のコメントに従い、ラッパー経由で import OmniP2P from './v3.js' できます。

クイックスタート

これだけで、E2EE チャット + 音声/動画通話 + 永続保存 + 共同編集がすべて動きます。

<script src="https://cdn.jsdelivr.net/gh/2chkakinie-arch/OmniP2P@main/v3.js"></script>
<script>
(async () => {
  // 1) インスタンス生成 (鍵は自動生成 or 既存の nsec/hex を渡す)
  const omni = new OmniP2P({
    relays: [
      'wss://relay.damus.io',
      'wss://nos.lol',
      'wss://relay.nostr.band',
      'wss://nostr.wine'
    ],
    // secretKey: '<64-hex or nsec...>'   // 省略で新規生成
  });

  await omni.start();               // リレー接続 + heartbeat 開始

  // 2) ルームに参加 (合言葉ベース、E2EE 自動)
  const room = await omni.join('my-secret-room', { password: 'p@ss' });

  // 3) メッセージ (WebRTC 優先、ダメなら Nostr で必ず届く)
  room.on('message', (msg, peer) => console.log('recv', msg, 'from', peer));
  room.send({ hello: 'world' });

  // 4) 音声/動画通話
  const call = await room.startCall({ audio: true, video: true });
  call.on('track', (stream, peer) => {
    const v = document.createElement('video');
    v.srcObject = stream; v.autoplay = true; document.body.appendChild(v);
  });
  // 画面共有に差し替え:
  // await call.shareScreen();
  // 録画して暗号化保存:
  // const rec = call.record(); ... const url = await rec.stopAndStore();

  // 5) 永続保存 (暗号化 KV / Blob DB)
  const ref = await omni.storage.put('note:1', { text: 'hi' }); // -> 参照
  const val = await omni.storage.get('note:1');                 // -> {text:'hi'}
  const url = await omni.storage.putBlob(fileBlob);             // -> 公開URL

  // 6) CRDT 共同編集
  const doc = room.doc('shared');       // MiniCRDT (Yjs 互換 API)
  const ymap = doc.getMap('state');
  ymap.set('count', 1);
  doc.on('update', () => console.log(ymap.toJSON()));
})();
</script>

比較軸従来スタックOmniP2P v3
導入 npm install × 数十、webpack 設定、TURN サーバー構築… <script> 1 行、ビルド不要、サーバー不要
シグナリング 自前のシグナリングサーバーを運用 公開 Nostr リレーを利用、サーバーゼロ

ℹ️ ストレージの信頼性: 書き込みはまず 1 本速い先へ投げ、 裏で Blossom → NIP-96 → IPFS ゲートウェイの順に非同期冗長化されます。 読み出しはコンテンツアドレスなので、どのプロバイダからでも同じ結果が得られます。

プラガブルアダプタ (差し替え / 増強)

コアはミニマルに保ちつつ、必要な機能だけをアダプタとして CDN から遅延ロードできます。 すべてのアダプタは、対応する SDK が読み込まれていればそれを使い、なければ内蔵実装にフォールバックします。 依存を強制しません。 

// 1) 明示的な遅延ロード (推奨)
await OmniP2P.loadAdapter('gundb');   // GunDB.js  リアルタイムグラフ同期
await OmniP2P.loadAdapter('matrix');  // Matrix.js Megolm E2EE 永続ルーム
await OmniP2P.loadAdapter('yjs');     // Yjs.js    本物の Y.Doc プロバイダ
await OmniP2P.loadAdapter('waku');    // Waku.js   並列匿名搬送 + Store 永続

// 2) 既にロード済み SDK と差し替える場合
OmniP2P.use(OmniP2P.adapters.Yjs);
OmniP2P.use(OmniP2P.adapters.Waku,   { pubsubTopic: '/omnip2p/1/msg/proto' });
OmniP2P.use(OmniP2P.adapters.Matrix, { homeserver: 'https://matrix.org' });
アダプタ名役割ロード元
MiniCRDTコア同梱の軽量 CRDT (Yjs 互換シェイプ)内蔵
Irisソーシャルグラフ (フォロー/DM 等)内蔵
Yjs本物の Y.Doc プロバイダ外部遅延
AutomergeAutomerge 用プロバイダ外部遅延
MatrixMatrix Megolm E2EE 永続ルーム外部遅延
GunDBリアルタイムグラフ同期 DB外部遅延
OrbitDBIPFS 上の P2P DB外部遅延
Waku並列匿名搬送 + Store 永続外部遅延

外部アダプタはすべて https://cdn.jsdelivr.net/gh/2chkakinie-arch/OmniP2P@main/<Name>.js から自動取得されます。

設計

  • Nostr の署名/検証は WebCrypto では secp256k1 が無いため、純 JS secp256k1 (schnorr, BIP-340) を内蔵実装。SHA-256 は WebCrypto。
  • E2EE の X25519 は純 JS 実装 (RFC 7748) を内蔵。HKDF / AES-GCM は WebCrypto。
  • すべて tree-shake 不要の単一クロージャ。グローバル汚染は OmniP2P のみ。
  • 大規模対応: リレーは多重接続 + レース、WebRTC はmesh + relay-mesh ハイブリッド、保存はコンテンツアドレス指定 + マルチプロバイダ冗長化。

FAQ

Q. サーバーは何も要らないのですか？

A. はい。シグナリングは公開 Nostr リレーで、永続化は Blossom / NIP-96 / IPFS ゲートウェイで行います。自前サーバーは不要です。TURN が必要な超厳しい NAT 環境のみ、任意で iceServers に指定してください。

Q. ブラウザ以外でも動きますか？

A. UMD 対応なので Node / Deno / Bun でも require / import で動きます (WebCrypto / WebRTC 相当の polyfill が必要な場合があります)。

Q. E2EE の鍵管理はどうなっていますか？

A. ルームの合言葉から HKDF でルーム鍵を導出し、ピア間は X25519 ECDH で共有鍵を作ります。すべての通信 (WebRTC / Nostr) と保存 (Blossom) の暗号化には AES-GCM 256 に AAD を付けて使用します。

Q. Nostr リレーが全部落ちたらどうなりますか？

A. 一度 WebRTC が確立していれば、そのまま P2P 通信は継続します。加えて「最低 1 本維持」フェイルセーフとスタガー再接続により、瞬断は極小化されます。

Q. コアだけ使いたいのですが、Yjs / Matrix / Gun は読み込まれますか？

A. いいえ。それらは OmniP2P.loadAdapter() を呼んだ時にはじめて CDN から取得される遅延ロードです。コアの初期バイトサイズには含まれません。
